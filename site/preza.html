Введение

Цель: создать открытый и воспроизводимый способ анализа текста и построения визуализаций для крупного литературного произведения.

Задачи:

  • разработать понятный формат визуализаций;
  • автоматизировать сбор и подготовку данных из текста;
  • построить наглядные визуализации;
  • опубликовать результат в открытом доступе.

Система разработки на телефоне

Инструменты:

  • Markor — черновики и заметки.
  • Termux — рабочая среда на Android.
  • GitHub — хранение и публикация проекта.
  • PI — помощник при написании кода.
  • Python — обработка текста и данных.
  • JS — веб-витрина визуализаций.

Сбор данных

Пайплайн:

  • нормализация текста (UTF-8, единые кавычки и тире);
  • разбиение на токены;
  • разбиение на предложения, абзацы, главы и диалоги;
  • лемматизация слов;
  • выделение имен собственных и их связей в тексте;
  • подсчет метрик сложности и тональности;
  • вывод данных в таблицы JSON и CSV.

Визуализация

  • Визуализации сделаны на JavaScript без API и внешних серверов.
  • Проект выложен в GitHub и запускается локально как статический сайт.

Реализация кода при помощи агента.

Реализация и выводы

Проект показывает полный конвейер: от текста до визуализаций и статической витрины. Все артефакты хранятся в outputs/<book_id>/ и экспортируются в site/public/data/.

Следующие шаги — улучшение выделения именованных сущностей, проверка персонажей, сравнение изданий и масштабирование на корпус.
