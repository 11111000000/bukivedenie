# AGENTS

Краткие правила для работы агентов в этом репозитории.

## Роли

- `pi` — читает, планирует, пишет тесты и короткие патчи-черновики. Код не запускает.
- `opencode` — вносит изменения, запускает тесты, build и smoke, собирает артефакты.

## Главное

- Все тексты и комментарии для пользователя — на русском.
- Изменения держать маленькими и локальными.
- Сначала failing test, потом минимальный фикс, потом проверка.
- Для визуализаций всегда сохранять артефакты smoke: `report.json`, `api.json`, `console.log`, HTML и скриншоты.
- Если есть shell `nix develop --impure`, использовать его как основной путь.
- Если нужен браузерный smoke, не ломать browserless fallback.

## Минимальный цикл

1. Прочитать нужные surface-файлы.
2. Добавить или обновить тест.
3. Внести минимальный фикс.
4. Войти в `nix develop --impure`.
5. Запустить нужную проверку: `pytest`, `cd frontend && npm test`, `cd frontend && npm run build`.
6. Запустить smoke: `make ui-smoke` или `nix run --impure .#smoke`.
7. Посмотреть `artifacts/ui-smoke/` и `logs/`.

## Что важно для окружения

- Shell должен давать: `python3`, `pytest`, `nodejs_22`, `chromium`, `git`, `curl`, `make`.
- Для shell можно использовать `direnv allow` один раз.
- Предпочитать инструменты из Nix, а не системные бинарники.

## Артефакты

- Основной каталог smoke: `artifacts/ui-smoke/`.
- Внутри должны быть: `report.json`, `api.json`, `console.log`, `html/*.html`, `screens/*.png`.
- Имена файлов должны быть безопасными и короткими.

## Файлы-ориентиры

- `agents/pi.md` — чеклист для `pi`.
- `agents/opencode.md` — чеклист для `opencode`.
- `scripts/ui_smoke.mjs` — канонический smoke-раннер.
- `scripts/ui_smoke.sh` — обёртка, которая при необходимости поднимает бэкенд.
- `frontend/tests/` — юнит-тесты и тесты smoke-хелперов.

## Контракт

- Если меняется публичное поведение, это надо зафиксировать тестом и кратко описать рядом с кодом.
