# Планы по параллельной отладке и улучшению UI

Задача: привести интерфейс к качественному SPA-поведению, обеспечить корректную работу всех визуализаций (Heatmap, Character Network и т.д.), сделать удобный адаптивный UX и организовать автоматизированную отладку и верификацию.

Подход: разделить работу на максимально параллельные субзадачи (субагенты), сначала поднять автоматизированную диагностику (smoke), затем локализовать и исправить конкретные проблемы, параллельно провести UX-рефакторинг. Всё документируется в `docs/plans/`.

Структура планов:

- `01-automated-debug.md` — автоматизированная отладка и расширение smoke
- `02-subagents.md` — описание субагентов (роли, входы, выходы, команды)
- `03-implementation-plan.md` — пошаговый план реализации и минимальные патчи
- `UX-design.md` — UX-цели, требования к адаптивности и примечания по стилю
- `checklist.md` — чеклист действий и критерии успеха

Далее — подробные планы по отдельным файлам.
