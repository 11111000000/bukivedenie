# Быстрый анализ результатов — чехов-письмо.txt

Дата: автоматическая

В данном документе собраны наблюдения по текущим экспортируемым данным из анализа текста "чехов-письмо.txt" и идеи, какую информацию можно из них извлечь.

1. Какие файлы генерируются (местонахождение):
   - outputs/чехов-письмо/
     - tokens.csv — частота токенов (топ токенов + per_1k)
     - frequency_dictionary.csv — аналог tokens.csv (частотный словарь)
     - characters.csv — список обнаруженных персонажей с контекстами
     - character_freq_by_chapter.csv — частота упоминаний персонажей по главам
     - cooccurrence_edges.csv — рёбра ко-встречаемости персонажей (weight = число контекстов)
     - sentiment_by_chapter.csv — простая оценка тональности по главам (лексиконный метод)
     - chapters_summary.json — разбивка на главы/абзацы/предложения (метаданные)
     - complexity_metrics.json — метрики сложности текста (нейтральные показатели)
     - hapax.csv — список hapax (единичных токенов)
     - run_metadata.json — параметры и время запуска

2. Ключевой контент и что из него можно получить:

   a) tokens.csv / frequency_dictionary.csv
      - Что содержит: стоп-слова и частые слова (токены) с частотами и нормированными скоростями (per_1k).
      - Что можно получить:
        - Частотный профиль стиля (частые служебные слова vs. лексика).
        - Выявление ключевых тем/лексики (по частоте контентных слов: "наука", "солнце", "луна" и т.п.).
        - Сравнение с другими текстами (лексический профиль).
        - Генерация облака слов, частотных списков для дальнейшего анализа.

   b) frequency_dictionary.csv (аналогично токенам)

   c) characters.csv
      - Что содержит: кандидаты‑персонажи (name, occurrences, context_sample и т.д.).
      - Что можно получить:
        - Основной список действующих лиц произведения (но требует ручной чистки/фильтрации).
        - Статистику упоминаний (кто наиболее упоминаем — вероятно главный герой).
        - Контекстные выдержки для ручной проверки идентичности персонажа (контекст_sample).
        - Можно строить временную линию упоминаний (offset → position in text) и наблюдать динамику появления.

   d) character_freq_by_chapter.csv
      - Что содержит: по‑главную разбивку упоминаний персонажей
      - Что можно получить:
        - Где именно (в каких главах) появляется персонаж — полезно для построения сюжетной карты.
        - Можно обнаружить появления эпизодических персонажей (единичные упоминания).

   e) cooccurrence_edges.csv
      - Что содержит: пары персонажей и вес ребра (сколько раз оба встретились в одном контексте)
      - Что можно получить:
        - Социальная сеть персонажей (graph) — наиболее тесные связи указывают на прямое взаимодействие/общий контекст.
        - Кластеры/подсети: кто чаще вместе появляется — семейные/социальные группы.
        - Центральность: кто наиболее «связан» — главный актёр.
        - Можно использовать вес и число глав (num_chapters) для фильтрации значимых связей.

   f) sentiment_by_chapter.csv
      - Что содержит: суммарная/средняя тональность по главах
      - Что можно получить:
        - Эмоциональный профиль сюжета по эпизодам (где текст позитивен/негативен).
        - Сопоставление пиков тональности с изменением частоты упоминаний персонажей.

   g) chapters_summary.json
      - Содержит структурированную разбивку на главы/абзацы/предложения с offset'ами
      - Что можно получить:
        - Контекст для всех упоминаний (быстро перейти к оригиналу по offset).
        - Использовать для визуализации расположения упоминаний по тексту.

   h) complexity_metrics.json
      - Примерные метрики сложности текста: средняя длина предложения, словарная насыщенность и т.п.
      - Что можно получить:
        - Статистики для лингвистического анализа и сравнения с другими авторами/текстами.

   i) hapax.csv
      - Список токенов, встречающихся только один раз — полезно для поиска редких имен, уникальных слов и шумов.

   j) run_metadata.json
      - Что содержит: время запуска, параметры, окружение
      - Полезно для воспроизводимости и аудита.

3. Замечания по качеству/ограничениям
   - NER — эвристический: ищет последовательности заглавных слов и применяет стоп‑лист + фильтр по наличию lowercase варианта. Это даёт много ложных срабатываний (навёл дополнения в стоп‑лист), но при этом может пропускать имена с вариациями/в падежах (Иван/Ивана).
   - character_freq_by_chapter и cooccurrence опираются на тот же NER, поэтому ошибки NER напрямую влияют на сеть отношений.
   - Лемматизация/нормализация не применяется повсеместно (pymorphy2 может быть отключён), поэтому формы имён в падежах ведут к раздвоению.
   - Отсутствует надёжное объединение альясов/сокращений (Петров vs П. Петров) — можно улучшить ручными правилами или кластеризацией по контекстам.

4. Быстрые рекомендации по дальнейшему использованию
   - Сделать ручную или полуавтоматическую фильтрацию characters.csv (минимум 2 вхождения или ручная проверка).
   - Свести формы имён через простую нормализацию (удаление падежных окончаний) или через pymorphy2 при доступности.
   - Построить граф из cooccurrence_edges.csv и визуализировать в Gephi/NetworkX/vis.js.
   - Сопоставить тональность и частоты для выявления кульминаций/смен настроений.
   - Использовать chapters_summary.json для плота распределения упоминаний вдоль текста.

5. Что можно добавить в отчёт (phots)

   - Сопоставить упоминания персонажей и sentiment_by_chapter для выявления событий.
   - Предложить правила объединения форм имён (например, сопадение лемм или минимальное расстояние в тексте).
